pnpm v11 `allowBuilds` workspace hijack: running `pnpm install -g <pkg>` with cwd=/root (or any dir lacking a pnpm-workspace.yaml) causes pnpm to CREATE a `pnpm-workspace.yaml` there holding `allowBuilds:` config with placeholder values ("set this to true or false"). Side effect: that dir becomes a workspace root. Any `pnpm install` in a subdir resolves UP to it; if the root `package.json` is empty `{}`, pnpm prints "Already up to date" and installs NOTHING in the subdir — no node_modules created. Diagnose with `pnpm install --ignore-workspace` (works) vs plain (no-op). Fix: allowBuilds is REJECTED in global config (~/.config/pnpm/config.yaml) — pnpm requires it in a project-level pnpm-workspace.yaml. Put it in the actual project dir (project-scoped, no hijack), delete the stray /root/pnpm-workspace.yaml + orphan /root/pnpm-lock.yaml + /root/node_modules.
